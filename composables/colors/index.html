Details

  • class: red, green, blue, etc.
  • settings: color-strength, ambient-mix
  • defaults: none
composable many classes

Examples

Give elements color by adding a color class:

<button>
  A button!
</button>

<button class="red">
  A button!
</button>

Project colors are generated from one parameter, the @base-color.

base-color

Here, it's given to you. But in your own project you can define it to be whatever you want.

Colors

Every Cavepaint project supports a 36-color palette, based on the base-color, each its own composable class:

Since the palette uses parametric design, each color is a bit different depending on the base color. If your project switches to a new base color, it will get a new 'cobalt' too.

Project colors

Four project colors are generated from the base color:

      action
      reaction
      ambient
      default

Semantic colors

Four aliased semantic colors are also chosen:

      info
      warning
      success
      failure

.light and .dark

Color classes compose with the .light and .dark classes:

<button class="light grapefruit">
  A button!
</button>

<button class="dark grapefruit">
  A button!
</button>

.texture

Colors compose with .texture:

A cream headline

<h3 class="cream text-padding">
Wow!
</h3>

A textured cream headline

<h3 class="cream texture text-padding">
Wow!
</h3>

Patterns can be invoked in any color using the .pattern() mixin.

They aren't available out-of-the-box in the CSS version.
