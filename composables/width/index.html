Project widths

There are several special width classes for finding just the right width for your element:

  • .text-width: The base content width, based on the @text-measure variable.
  • .aside-width: A smaller supplemental width, golden to text-width
  • .page-width: The page width, like text-width and aside-width added together
  • .full-width: Makes an element full-width
  • .golden-width: Makes an element golden to its container. .reverse reverses.
So, aside-width is the width of this aside. Then text-width is the width of this text, and page-width is the width of this page.

Inner widths

Inner-width classes use padding to emulate an inner-width effect.

  • .inner-text-width: The base content width, based on the @text-measure variable.
  • .inner-aside-width: A smaller supplemental width, golden to text-width
  • .inner-page-width: The page width, like text-width and aside-width added together

This example page is already set to inner-text-width, so we'll use the smallest of the three, aside-width, for examples:

This is just a regular red paragraph with responsive and padding composables added to it.

<p class="red responsive padding">
    This is just a regular <strong>red</strong> paragraph with <strong>responsive</strong> and <strong>padding</strong> composables added to it.
</p>

The same element and composables, but with inner-aside-width added.

<p class="red responsive padding inner-aside-width">
    The same element and composables, but with <strong>inner-aside-width</strong> added.
</p>

This one instead has aside-width and margin-auto.

<p class="red responsive padding aside-width margin-auto">
    This one instead has <strong>aside-width</strong> and <strong>margin-auto</strong>.
</p>

The inner-x-width is great for keeping markup simple and using fewer containers.
