Finally, a CSS framework that's just right.

CSS frameworks have a Goldilocks problem.

The first type of framework is the ultra-thin CSS file. You download it, enjoy the small file size and codebase, and then it's all downhill from there. The tiny file eventually becomes a bloated, unorganized mess, or the preamble to a bloated, unorganized mess.

The second type of framework takes the opposite approach, including everything under the Sun. These frameworks will usually rely on advanced tools to chisel away at a CSS monolith, only keeping the classes a project is actively using.

Both of these approaches suffer from too little or too much. They're both wrong ideas about what makes CSS good for a project.

Cavepaint CSS is designed to be just right. Weighing in at around 35kb, it's not going to win any minimalist CSS prizes. On the other hand, there's no fat to trim, no esoteric classes to pull out and no workflow to configure. There's no tree-shaking or gzipping to be done. It's just normal CSS readable by both humans and machines.

Cavepaint doesn't get bloated because it already has so much of what you need. It doesn't need a chisel because it doesn't start out ridiculously huge.

Cavepaint CSS is the CSS you need, for any project, right now.
