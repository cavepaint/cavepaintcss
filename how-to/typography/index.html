Typography in Cavepaint

Cavepaint is heavily based on typography.

The widths and breakpoints in the project are based on three widths, which are based on text measure (how many characters occur on a line of text).

The first width is text-width, which is designed to give an element a width that will produce correct text measure for body copy. All of the text on this page is set to that width.

The next width is aside-width, which is the golden complement of text-width. The two together make up the third width, page-width. That's the width of this page's full content.

aside-width

These three widths also support their inner-width counterparts: inner-aside-width, inner-text-width and inner-page-width.

All three widths are based on the @text-measure variable which approximates how many characters run per line.

Responsiveness breakpoints are also based around these widths

Helper classes

These classes can help get text looking correct:

  • .small
  • .big
  • .huge
  • .text-center
  • .text-padding
  • .text-margin
  • .text-shadow
  • .bold
  • .underline

Fonts

Cavepaint's font stacks are designed to use system fonts, or in other words designed to already be on the user's computer.

  • .serif
  • .sans-serif
  • .monospace

Like any CSS, Cavepaint will work fine with web fonts.
