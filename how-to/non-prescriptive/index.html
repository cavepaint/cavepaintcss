Avoiding a certain look

For some web frameworks, the whole point is standardizing and looking alike.

We get it. We're just not one of those frameworks.

Some web frameworks end up with a certain standardized look because of the way they're built. But it's usually fine because they're built around customizing. Still, a color slips through here and there, and you end up the same warning orange as seven hundred thousand other sites.

Cavepaint is built for the decentralized web, so we're decentralizing color palettes too. Cavepaint keeps non-prescriptiveness as a core developing concept, never assigning any particular color or look & feel to an element.

Check it out: Try a whole new set of parameters

Because the color scheme is generated using parametric design, another core concept, each color in each scheme is just a little bit different. The color "grapefruit" may be darker or redder depending on the input @base-color.

Cavepaint core concepts

  • compose small powerful ideas together
  • use parameters to generate design
  • avoid prescribing any look or feel
