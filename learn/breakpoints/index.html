Mobile-first

Breakpoints generally work with a mobile-first dynamic. The exception, appropriately, is "mobile" which is like saying "mobile-only" in terms of where and how it behaves.

A two-column composable will by default always appear as two columns, unless used with tablet or desktop.

Text-relative

Breakpoints aren't defined in prescriptive pixel widths. Instead, they're based on text measure, expressed in rem, and will change depending on text settings.

Responsive gaps/padding

The responsive helper modifies padding, margin and gutter composables to be responsive. Additionally, it also works on any grid gaps, so the space between columns will be adjusted for layout composables.

  • On mobile: ~40% of value
  • On tablet: ~60% of value
  • On desktop: Full value (same as no helper)

The responsive helper is distinct from the breakpoint helpers mobile, tablet and desktop although it may be used alongside them.

Responsive text

The responsive-text composable will make text smaller on mobile and larger on desktop. It's best used on the body element.
